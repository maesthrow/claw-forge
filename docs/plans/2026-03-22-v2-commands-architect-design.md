## 1. Переименование: оркестратор → архитектор

Везде в проекте заменить orchestrator/оркестратор на architect/архитектор:

| Что | Было | Стало |
|---|---|---|
| Папка | `agents/orchestrator/` | `agents/architect/` |
| SOUL.md | "Я оркестратор" | "Я архитектор AI-агентов" |
| AGENTS.md | упоминания оркестратора | архитектор |
| SKILL.md | описания | архитектор |
| orchestration.py | промпты | архитектор |
| deploy.py | маппинг orchestrator→main | architect→main |
| main.py | комментарии | архитектор |
| setup.py | путь к папке | architect |
| README.md | все упоминания | архитектор |

В OpenClaw агент остаётся `main`. Команда возврата — `/main`.

## 2. Система команд

| Команда | Действие | Где работает | CLI |
|---|---|---|---|
| `/list` | Список агентов | Архитектор | `python3 main.py list` |
| `/set <name>` | Переключить на агента | Архитектор | `python3 main.py switch --agent <name>` |
| `/rm <name>` | Удалить (с подтверждением) | Архитектор | `python3 main.py delete --agent <name>` |
| `/main` | Вернуться к архитектору | Любой агент | `python3 main.py switch --agent architect` |
| `/new` | Новая сессия | Нативная OpenClaw | — |

Команды обрабатываются через SOUL.md → skill claw-forge → exec python3 CLI.

## 3. Приветствие

### Архитектор (SOUL.md)
Правило, не шаблон:
```
При первом сообщении в сессии ВСЕГДА:
1. Представься — кто ты (ClawForge, архитектор AI-агентов)
2. Кратко скажи что умеешь (создавать, подбирать, управлять агентами)
3. Покажи список доступных команд: /list, /set, /rm, /new
4. Спроси что пользователь хочет сделать
```

### Создаваемые агенты (генерируется developer'ом)
```
При первом сообщении в сессии ВСЕГДА:
1. Представься — кто ты (имя + краткое описание роли)
2. Покажи: /main — вернуться к архитектору, /new — новая сессия
```
